# README Composition Aid

This file is not a mandatory template. Use only the parts justified by the project.

```md
# Project Name

One or two sentences defining what the project implements, the stack or level of abstraction, the task, and the meaningful distinction.

A short paragraph explaining the main design priority or tradeoff.

Optional: a short explanation of implementation size or the few key files, when this helps the reader understand the project.

## Quick start / Usage / Example usage / To run

Prerequisites or installation commands placed close to the first run.

```bash
exact command
```

A short explanation of what the command loads, creates, runs, or saves.

```text
real output or confirmed success signal
```

Optional short comment after the result.

## Project-specific scenario

Only add sections required by the actual project, such as training, inference, datasets, tests, hardware variants, benchmarks, todos, references, or license.
```

Do not automatically add:

- Features;
- Architecture;
- Project structure;
- Installation;
- Configuration;
- Contributing;
- FAQ;
- Roadmap;
- Acknowledgements;
- License.

Any of these may be valid, but only when the project itself justifies them.
